¿De qué va esto?

El Web Games Day reúne en un día las novedades en la plataforma Web que impulsan el desarrollo y la distribución moderna de videojuegos.

Se trata de un evento para todos los entusiastas de videojuegos, técnicos o no, que se interesen en la plataforma universal, libre y carente de fricción que es la Web.

Este Web Games Day nace como la culminación de una pequeña colaboración piloto entre el grado de programación de video juegos de la Facultad de Informática de la UCM y el equipo de DevRel de Mozilla. Nuestra intención es la de mostrar cómo la Web apoya y promueve tecnologías clave en el desarrollo y la distribución moderna de video juegos.

A esta colaboración se unen las comunidades MadridJS, HTML Spain y Video Game Army con el fin de difundir el evento, que se presta de interés no sólo para entusiastas de los video juegos sino para todos aquellos que disfruten o trabajen con tecnologías Web.

Building VR experiences

WebVR es una API para crear experiencias de realidad virtual en el navegador. También contamos con algunas herramientas que nos permiten crear tales experiencias de manera mucho más fácil y más productiva, como A-Frame, un framework de realidad virtual de código abierto. En esta charla descubriremos cómo usar A-Frame y algunas de sus posibilidades.

The Web Audio API

Audio y tecnologías web... no tiene por qué ser siempre la típica y molesta música de fondo que comienza a sonar cuando abres una pestaña.

La API de Web Audio es extremadamente potente, podemos crear sonidos, cargarlos, analizarlos y modificarlos de miles de maneras distintas.

Esta es una charla en profundidad acerca de los diferentes métodos de la API y, no sólo eso, sino también acerca de cómo podemos integrarlos en nuestro proceso de desarrollo. ¿Cuáles son las mejores técnicas a la hora de cargar y manipular sonido? Esta charla intenta explorar precisamente eso, con la idea de que te sientas más seguro sabiendo qué métodos usar cuándo.

WebAssembly Fundamentals

Esta presentación presenta WebAssembly (WASM), un nuevo formato para los programas en la Web. Como alternativa a JavaScript, WASM puede correr órdenes de magnitud más rápido —casi tan rápido como C— aun funcionando en cualquier navegador.

En esta charla aprenderás: 1) ¿Qué es WASM? 2) ¿Por qué necesitamos WASM? 3) ¿Cómo funciona WASM? 4) ¿Cómo puedes usar WASM hoy? 5) ¿Cuál es el futuro de WASM?

Dan Callahan es ingeniero de software en el equipo de relaciones con desarrolladores de Mozilla, donde se encarga principalmente de Servo (el navegador con alta paralelización), y de los complementos de Firefox. Anteriormente, Dan trabajó en el proyecto Persona de Mozilla, un intento de reemplazar las contraseñas con un sistema de criptografía pública amigable y construido sobre estándares de la Web abierta.
